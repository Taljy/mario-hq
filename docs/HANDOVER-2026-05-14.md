### 4.1 · Watchlist ist bewusst generisch
`watchlist.json` enthält NICHT Marios echte TradingView-Liste, sondern eine generische Platzhalter-Liste. **Das ist bewusst entschieden (14.5.2026).** Mario will die Wirtschaftsseite später nochmal überarbeiten sobald er Praxiserfahrung hat. Spec §4.1 hat einen Vermerk dazu, das echte Schema bleibt dort als Referenz für später. → Slice 4.4 baut die UI auf der generischen Liste, das ist OK.

### 4.2 · Glassnode-Evaluierung offen
Mario ist auf Glassnode gestossen (On-Chain-Analytics, Industrie-Standard). Er ist **zahlungsbereit-offen** (Studio-Subscription ~$39/Monat, API kostenpflichtig). Soll eine eigene Strategie-Runde bekommen NACH Slice 4.4 · dann mit echter Nutzungs-Erfahrung der Phase-4-Trading-Indikatoren. Dokumentiert in `_pendenzen.md` Offene Fragen.

### 4.3 · Trading-Daten-Tiefe vs Asset-Tiefe
Ungeklärte strategische Frage: mehr Indikator-Tiefe (Glassnode-Richtung) oder mehr Asset-Breite (Watchlist-Richtung)? Gehört in die Glassnode-Strategie-Runde.

### 4.4 · Vercel-Doppel-Projekt · welches ist kanonisch

## 5 · Bekannte Production-Issues (nicht blockierend für Slice 4.4)

### 5.1 · Vercel blockt Binance-API
Auf Production zeigen 4 Trading-Indikator-Cards "FALLBACK · BINANCE OFFLINE" (Funding Rates, Open Interest, Long/Short Ratio, Coinbase Premium). Lokal funktioniert alles. Wahrscheinlich Geo-/IP-Block von Binance gegen Vercel-Server-IPs. → **Mini-Reparatur-Slice** geplant. Lösungswege: Edge-Function-Proxy, alternative API (Bybit/OKX), oder pragmatisch akzeptieren. Slice 4.4 (Watchlist) ist NICHT betroffen · Twelve Data und CoinGecko haben keine Geo-Blocks.

### 5.2 · Vercel-Doppel-Projekt
Siehe 4.4 oben. → **Mini-Hygiene-Slice** geplant.

### 5.3 · Twelve Data Free Tier · Indices + Commodities limitiert
Beim Slice-4.3-Test bestätigt: Twelve Data Free Tier liefert KEINE Indizes (SPX, NAS100, DAX, SMI) und BRENT nicht, WTI nur teilweise. Erwartet · Fallback-Strategie ist in Slice 4.5 zu klären (Alpha Vantage oder weglassen). Der watchlistAggregator behandelt fehlende Items graceful (ist_live: false).

## 7 · Etablierte Arbeitsweise (bewährt über Phase 2.2-4.3)

- **Spec-First:** Phasen-Spec ins Repo committen bevor Slices gebaut werden
- **Plan-First:** Claude Code zeigt Plan vor jedem nicht-trivialen Edit
- **Slice-Pattern:** pro Slice ein feat-Commit + ein docs-Commit
- **Foundation-First:** der erste Slice einer Phase räumt das technische Risiko ab
- **Pattern-Klon:** jeder Slice baut auf einem Vorbild auf
- **Verifikation:** Build grün → Light/Dark/Mobile → Console → Vercel HTTP 200
- **GO-Prompt-Stil:** claude.ai schreibt ausführliche strukturierte GO-Prompts, Mario kopiert sie in den Claude-Code-Chat
- **Stil-Treue:** Schweizer Hochdeutsch, du-Form, Em-Dash, Mittelpunkt, DRG-Tokens, keine Floskeln

**Stolpersteine die immer gelten:**
- Worktree-Falle · `cd /Users/mariomacstudio/Developer/mario-hq && ...` chainen
- Build-Test vor Push · Pflicht
- ENV-Vars: lokal in .env (gitignored) + beide Vercel-Projekte · Redeploy nötig
- TypeScript strict · keine any
- Optionen-Format bei Klärungsfragen: 1 · 2 · 3 nummeriert, Empfehlung explizit kennzeichnen
